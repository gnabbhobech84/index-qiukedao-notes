# index-qiukedao-notes

一个用于归档和发布多个独立 HTML 页面的仓库。这里存放的页面内容不针对特定域名或网站，仅作为静态页面的集中存储与展示。

## 项目简介

本项目主要目的是将多个独立的 HTML 页面统一归档，方便通过 GitHub Pages 或其他静态托管服务进行访问。每个页面均为独立文件，彼此之间没有依赖关系，适合存放一些临时性的笔记、演示页面、工具界面或信息展示页。

## 目录说明

仓库根目录下存放的 `.html` 文件即为可独立访问的页面。目录结构如下：

```
.
├── index.html          # 仓库入口页面（如有）
├── page1.html          # 示例页面
├── page2.html          # 示例页面
└── README.md           # 本文件
```

页面文件命名应保持简洁、有描述性，避免使用特殊字符或空格。

## 页面归档说明

- 每个 HTML 文件均为独立、自包含的页面，建议将样式、脚本内联或通过相对路径引用。
- 页面内容可以是笔记、工具、原型、演示等，不限于特定主题。
- 若页面涉及外部资源（如图片、字体），请确保资源链接有效或一并托管于本仓库。

## 维护说明

- 新增页面：直接在仓库根目录下添加 `.html` 文件，并更新本 README 中的目录说明（可选）。
- 删除页面：移除对应文件，并更新目录说明。
- 修改页面：直接编辑对应 HTML 文件，注意保持页面独立性。
- 建议为每个页面添加清晰的标题和描述，便于识别。
